Integrations

Email digests · Slack alerts · Google + Outlook calendar sync. All optional — the tool works without any of them.
Every integration is off by default. It turns on when the relevant secret is set in GitHub repo settings. Skip any you don't need.

Weekly email digest optional · 10 min

Friday 17:00 UTC: tools/send-digest.js reads the week's progress log, builds an HTML email of shipped items + closed checkpoints + streak, sends via Resend.

  1. Sign up at resend.com (free tier: 3000 emails/month, 100/day).
  2. Verify a sending domain (their docs walk through DNS records). Without a verified domain, you can only send from onboarding@resend.dev to your own inbox — fine for a pilot.
  3. Create an API key. Copy it.
  4. In the repo: Settings → Secrets and variables → Actions. Add the values below.
  5. Trigger a dry-run: Actions → friday-digest → Run workflow, check "Dry run" on. It'll log the payload without sending.
NameTypeValue
RESEND_API_KEYSecretre_xxxxxxxxxxxxxxxx
DIGEST_FROMVariableresearch-pm <noreply@yourdomain.com>
DIGEST_TOVariableyour-email@example.com, pi@lab.edu
DIGEST_CCVariable (opt)collaborator@...

Source: tools/send-digest.js · Schedule: .github/workflows/friday-digest.yml. Substitute SES or Postmark — the script is ~100 lines, easy to port.

Slack alerts optional · 5 min

Posts to a team channel when a blocker is filed (via the New Blocker issue form). Can be extended to other events easily.

  1. In Slack: Apps → Incoming Webhooks → Add to Slack. Pick the channel (typically #lab-blockers).
  2. Copy the webhook URL.
  3. Repo Settings → Secrets → Actions → add SLACK_WEBHOOK_URL.
  4. Test: file a blocker via the website quick-log or New Blocker form. Within 10 seconds the message should land in your channel.
NameTypeValue
SLACK_WEBHOOK_URLSecrethttps://hooks.slack.com/services/T.../B.../...

Source: tools/notify-slack.js. Called from .github/workflows/parse-issue.yml when the blocker label is present. Skips silently if the secret isn't set.

Google Calendar sync optional · 20 min

Nightly at 06:00 UTC: tools/calendar-sync.js reads your Google calendars, writes calendar/YYYY-MM.md, commits back. The standup command then sees your real schedule when deciding today's calendar shape.

  1. Go to Google Cloud Console. Create a new project (name it "research-pm" — one-time).
  2. APIs & Services → Enable APIs → enable Google Calendar API.
  3. OAuth consent screen → External → fill app name, your email, scopes: .../auth/calendar.readonly, .../auth/calendar.events. Add yourself as a test user.
  4. Credentials → Create → OAuth client ID → Desktop app. Download the JSON. Note client_id and client_secret.
  5. One-time locally: run the bootstrap script to get a refresh token. 
    node tools/oauth-bootstrap-google.js
    Follow the prompt (opens a browser), grants consent, prints a refresh token.
  6. Repo Settings → Secrets & Variables → Actions:
    • Add secrets: GOOGLE_CLIENT_ID, GOOGLE_CLIENT_SECRET, GOOGLE_REFRESH_TOKEN
    • Add variable: GOOGLE_CALENDAR_IDS (comma-separated — primary or specific calendar IDs)
  7. Test: Actions → nightly-sync → Run workflow, with dry-run on. Should log this month's events.
NameTypeValue
GOOGLE_CLIENT_IDSecretfrom OAuth credentials
GOOGLE_CLIENT_SECRETSecretfrom OAuth credentials
GOOGLE_REFRESH_TOKENSecretfrom bootstrap script
GOOGLE_CALENDAR_IDSVariableprimary or primary,work@company.com

Source: tools/calendar-sync.js · Schedule: .github/workflows/nightly-sync.yml. Writes are silently skipped if the secret isn't configured, so it's safe to leave the workflow file committed.

Outlook / M365 sync optional · not yet implemented

Same pattern as Google, using Microsoft Graph. The script hook exists in calendar-sync.js; the API call implementation is Phase 4.5.

If Outlook is your primary calendar, open a feature-request issue and we'll prioritise it. Temporary workaround: share your Outlook calendar with a Google account, point GOOGLE_CALENDAR_IDS at it.

Writing back to your calendar planned · Phase 4.5

Phase 1 is read-only sync — the site and Claude see your calendar but don't modify it. Phase 4.5 adds write-back:

  • When you approve the Monday /week-plan, Claude creates events titled "Writing — TNF methods" on your work calendar, so colleagues see your protected blocks.
  • Scope will default to the work calendar only; personal stays read-only for safety.

Requires the calendar OAuth scope upgrade to .../auth/calendar.events (writable) — already included in the setup above in anticipation.
