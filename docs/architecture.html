How does Claude get instructions?

Claude Code reads these three things when it opens this repo:

FileDoes what
CLAUDE.md Loads the six principles, tone rules, priority order, data model. Every session starts with this.
.claude/commands/*.md One file per slash command. /standup, /week-plan, /digest, /retro, /newcheckpoint, /logwin, /one-on-one, /report, /move, /onboard.
.claude/settings.json SessionStart hook auto-runs /standup once per day. Permissions for calendar + email tools.

Promises made on paper → file on disk. When the deck says "/standup shows today's 3 + overdue + calendar shape," that structure lives in .claude/commands/standup.md. Claude reads it and produces that structure.

How do website changes reach Claude?

Not directly. Via the repo.

  1. You do something on the website (move a block, log a win).
  2. Phase 1: the action opens a pre-filled GitHub Issue. A GitHub Action parses it and commits the markdown change.
  3. Phase 2: the PWA commits directly via the GitHub REST API (OAuth device flow).
  4. Next time you open Claude Code, it reads the updated markdown.

Eventual consistency, not live. If you drag a block on your phone and immediately open Claude on your laptop, Claude sees it. There's no realtime socket between the two.

A day, end-to-end

1

9:00 — you open Claude Code

SessionStart hook fires. Checks if /standup has run today. It hasn't. Runs it. Claude reads week-plans/2026-W17.md, projects/*.md, progress/2026-04.md, calendar/2026-04.md, goals.md, people.md. Produces the standup card you see on the Standup page.

2

9:02 — you confirm and start writing

Protected 09:00–11:00 Deep block is on your calendar (Claude wrote it there when you approved this week's plan on Monday).

3

11:15 — new meeting lands on your work calendar

Nightly sync isn't running yet, but on next sync it sees the conflict with tomorrow's writing block. Re-shuffles the week, flags it in tomorrow's standup: "Wed writing block clobbered by new meeting — moved to Thu morning. OK?"

4

14:00 — on the bus, you tap the phone

Website in a browser (mobile-friendly). You log the passage on the Plan Board — "+ Log passage" opens a pre-filled GitHub Issue. Submit. Action runs, commits progress/2026-04.md.

5

16:15 — 1:1 with Sara starts

Agenda from meetings/2026-04-21-1on1-sara.md, drafted from her owes-me list + recent progress. You add notes during the meeting. On close, Claude extracts action items back into people.md (owes-me / I-owe updated).

6

Friday afternoon

/digest: names shipped items from the week's progress log. /retro: notes what slipped, one adjustment for next week. Both written to retros/2026-W17.md. Optional: emails a weekly collaborator report to Prof Smith (scheduled action, Resend/SES).

FAQ

Do I need a paid Claude plan?

Any Claude Code-supported plan works (individual or team). Claude Code itself is free to install. No API keys to manage separately — Claude handles auth.

What about my collaborators — do they need accounts?

No. Claude generates reports (markdown / email / slide-ready) that you send. The website is optional-access with a Cloudflare Access password gate if you want selective sharing.

Where does the data live?

In markdown files in a git repo, which you own. Nothing lives in a vendor database. If you stop using the tool, your project history is already portable.

What costs money?

Claude account (what you pay for anyway), GitHub (free for up to 3 private collaborators or unlimited public), domain if you want one. Cloudflare Access free tier covers up to 50 users. Calendar API is free. Email digests via Resend/SES free tier.

What can break?

Mostly integrations: calendar OAuth can expire (Claude re-prompts in next session); issue-form Action fails occasionally on malformed input (shows up as a comment on the issue). Core flow — markdown ↔ Claude chat — has no failure modes beyond git itself.

Is the website authoritative, or is Claude?

Neither. The markdown files are authoritative. Website and Claude are two clients on the same data.
