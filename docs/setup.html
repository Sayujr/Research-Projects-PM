Setup — one paste, one conversation

You already have Claude Code. Paste the prompt below once. Claude does the rest — clones the repo, checks the setup, starts the onboarding interview.
Prerequisites: a Claude account (any paid plan) and Claude Code installed. That's it. No git commands to memorise, no npm install, no env vars.
1

Open Claude Code on your laptop

Any folder — it doesn't matter where. Just run claude in a terminal. Claude Code is free to install from claude.ai/code. If you already use it, you're fine.

2

Paste this single prompt

Claude does every other setup step. It will ask permission before cloning or writing files.

Set up research-pm for me on this machine.

Steps:
1. Clone https://github.com/Sayujr/Research-Projects-PM.git into
   ~/research-pm (pick another location if that's taken; confirm with me).
2. cd into the cloned repo.
3. Read CLAUDE.md so you load the six design rules, the goal cascade
   (year → quarter → project → week → today), the priority order
   (deadlines > writing > unblock team > perishable lab > experiments
   > admin), and the tone rules.
4. Confirm the six design rules back to me in one short paragraph so I
   know you've loaded them.
5. Start the /onboard slash command — stage 1 (role + year goals). Run
   conversationally, one question at a time. Do not dump all 8 stages at
   once. Spread the remaining 7 stages over future sessions.
6. After stage 1, commit locally (push only if I ask). The SessionStart
   hook in .claude/settings.json will auto-fire /standup on future
   session starts.

Behaviour from here on:
- Every new session: /standup runs automatically via the SessionStart hook
  (checks .claude/last-standup). If the hook doesn't fire, run /standup
  manually.
- When I log work, use /logwin — it appends to progress/YYYY-MM.md with
  the date, project, note, and duration (if I gave one), then updates
  meta/task-history.json for duration learning. Never lose a duration.
- Decide on my behalf. Only ask when the choice is genuinely mine.
- Keep responses short unless I ask for depth. Evidence-based praise only.
  No "great work" without naming what actually shipped.
- Follow the output format specs in .claude/commands/<name>.md exactly
  — no ad-lib formatting.

Ready? Start with step 1.

This prompt is also stored in this repo at setup-prompt.md. If you want to tweak it before pasting, edit there.

Over the next few days, Claude runs the other 7 stages

One stage per session. Projects, team, typical week shape, task durations, calendars, full goal cascade, preferences. By end of week 1, the tool is calibrated to you.

  • Stage 2 — active projects (one-liners + collaborators + deadlines)
  • Stage 3 — team (postdocs / techs / students)
  • Stage 4 — typical week shape + lab hours
  • Stage 5 — task duration estimates (seeds duration learning)
  • Stage 6 — calendar platforms (Google + Outlook)
  • Stage 7 — goals broken into quarterly OKRs or narratives
  • Stage 8 — preferences (nudge hours, motivation tone, notification channels)

Website + Claude — do I need both?

No. Claude Code alone is enough. The website is a visual layer for:

  • Phone access to the dashboard
  • Collaborator view (your PI / co-authors)
  • Drag-drop Plan Board when a chat is clunky

If you live in Claude chat, you can skip the website entirely. Dashboards still regenerate into markdown.

Prefer the manual path? git clone ... && cd research-pm && claude works the same. The single-prompt path is for when you want to skip the setup narrative.
